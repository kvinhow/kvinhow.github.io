Quartus II 17.0 仿真教程

发表于 2018-04-05 | 分类于 FPGA | 阅读次数:
字数统计: 916字 | 阅读时长 ≈ 3分钟

关于最新版本quartus ii 17.0（我用的lite版）的仿真问题，在百度查阅很久+自己的摸索后才找到如何正确的仿真，写一个简短的教程希望可以帮助到学习FPGA的朋友。

工程配置

建立工程

填写工程信息

tips： 顶层设计实体名必须顶层文件名一致，不能使用中文（包括工程目录所在不要有中文），不能使用Verilog关键词或者与Quartus II设计库中的模块名称相同。

选择好工程目录，以及填写完工程名及顶层实体名后，直接NEXT默认到 Add files 页面

添加设计文件

如果有写好的设计文件，可以点击圆圈位置去文件资源管理器选择对应的文件，选择好后记得点击ADD。我这里从零开始，就直接NEXT跳过了。

选择芯片

我使用的是黑金AX301 FPGA 开发板是属于Cyclone IV E系列的，具体型号为EP4CE6F17C8。大家根据自己的开发板选择相应的型号即可。

EDA工具设置

在这里可以选择后面仿真所用的工具以及文件格式，也可以直接跳过，在后面再进行配置。这里我选择的是Modelsim-Altera和Verilog HDL，本文所述仿真也是基于此设置进行的。

工程信息

确认无误后点击Finish即可。

编写.V文件及激励文件的配置

创建Verilog HDL文件

如图所示创建Verilog HDL文件

当顶层实体不是之前在配置所填写的“top“时会出现以下报错，此时选中你所写的顶层文件同时按下”Ctrl+shift+J“即可将该实体设置为顶层实体。也可选中该文件点击——>project——>set as Top-Level entity。

将Verilog 代码写完保存时要注意文件名要与实体名相对应，不然在进行综合时会出现类似以下的错误

生成测试文件

仿真需要为所其写一个激励文件，因此我们要为当前顶层实体生成对应的激励文件。

当在processing看到如下内容时说明激励文件已经创建成功。

打开生成的激励文件，使用Ctrl+O的快捷键打开文件资源管理器，选择simulation文件夹打开modlesim选择对应的.vt文件（这里的顶层实体名是led_run，对应的激励文件名也是led_run)如果进入modlesim发现没有文件，在文件资源管理器的右下角更改为显示所有文件即可。打开文件如下图所示。

tips:复制文件中“led_run_vlg_tst”,这个将在后面的配置中起到至关重要的作用

EDA工具设置

按照下图进入setting界面找到“EDA Tool Settings”，选择“simulation”。

按照我给的图进行设置，点击红圈处test benches进行激励文件的配置

点击new新建配置

之前所复制的内容就是这里的“top level module in test bench”，关于这里的配置在图片中又详解，之前所生产的“led_run.vt”通过点击红圈处添加，注意一定要点ADD加入，点击OK即可，然后一直点击OK，关于EDA的设置到此就已经完成。

编写激励文件

关于激励文件的编写随便找一本Verilog的教材基本都会介绍，所以在此我就详细介绍了

仿真

在上述设置完成后进行“compile design”，编译无误后即可开始仿真。
